# FanOps — project notes for Claude

MOH FLOW FAN OPS: intelligent clip + cross-post engine. Pure-Python `src/` layout
(`src/fanops/`), console script `fanops`, Python 3.12–3.13.

## Commands

- Install: `pip install -e '.[dev]'` — extras: `[studio]` (Flask cockpit, imported lazily), `[transcribe]` (whisper CLI), `[compose]` (MoviePy produced-clip compositing — `fanops compose`, lazy + fail-open)
- Fast unit suite (CI `unit` job): `python -m pytest -q -m "not integration"`
- Integration suite (CI `e2e` job): `python -m pytest -q -m integration -rs` — needs real ffmpeg/ffprobe/whisper/espeak on PATH; skips locally when absent; CI sets `FANOPS_REQUIRE_E2E=1` so a skip fails
- Lint: `ruff check .` (pyflakes F + pycodestyle E only)
- Studio dev server: `fanops studio` (localhost:8787; requires `[studio]` extra)
- Browser ingestion (no Finder): the Studio **Run** tab **Upload video** form streams raw video into `01_inbox/` (validated: video ext, traversal-safe `secure_filename`, inbox-bound resolve, atomic `.uploadpart`→`os.replace`; 2 GiB `MAX_CONTENT_LENGTH` cap) → click **Ingest inbox** to catalogue it. `actions.save_uploads` owns the contract; an oversize body re-renders the panel at HTTP 200 ("too large") since htmx 2.x drops non-2xx swaps.
- Approval lifecycle (no auto-publish): a crossposted post is BORN `PostState.awaiting_approval` — `publish_due`/`publish_now` iterate only `queued`, so NOTHING publishes (even on a live backend / daemon) until the operator approves it. The Studio **Review** tab is the approve worklist (per-surface checkboxes → batch **Approve selected** / **Reject selected**; `Ledger.approve_post` promotes awaiting→queued). Per-post-per-surface scheduling is first-class (P1): approving an UNTIMED / stale-past post no longer bumps it to **now** (which silently published it on the next `publish_due`) — it lands a deterministic **strictly-future suggestion** (`views.suggest_time` = `crosspost.surface_time` at `index=0`, never a 40-min stagger), so it sits in `queued` for the lead window instead of firing instantly; a still-future operator-set time is preserved verbatim. Every editable Review surface AND Schedule row offers **Use suggested** (one-click accept the suggestion) and **Clear time** (`actions.clear_time` — on a queued post it FIRST un-approves back to Review, THEN clears, so a post is never left `queued`-and-timeless). The **Schedule** tab is the approved-posts bucket cockpit (shows each post's Postiz integration; per-row Move / Use suggested / Clear time / Publish now / Send back to Review; **Reschedule all** re-spreads via `crosspost.surface_time` — the 40-min auto-stagger is reachable ONLY here, never imposed on an individual post). The **Posted** tab is the all-time shipped library (live URL + lift) + **Post again** (`actions.repost_post` spawns a fresh awaiting_approval repost of the same clip with an epoch-suffixed id — honors fan-accounts-repost-freely, NOT a supersede). `queued` now means "approved + scheduled"; `rejected` is an operator discard. Mirrors the M3/M4 stitch approve/release gate.
- Onboard + go live (no env vars / CLI / JSON): the Studio **Go Live** tab connects Postiz, **adds an account** (handle + platforms), maps **each (handle × platform) channel** to its own Postiz integration id (`accounts.json` `integrations` is per-platform — a handle's IG and TikTok are different integrations; a legacy single `account_id` stays the fallback), and flips dryrun↔live behind a confirm (dual-writes `.env` + `os.environ`; the API key is write-only — never rendered). `go_live` is the ONLY setter of `FANOPS_LIVE=1` — gated on accounts-valid → ≥1 live-ready channel → past-due-backlog gate → explicit confirm — and it NEVER writes `FANOPS_POSTER` (D12: per-channel `accounts.json` routing via `set_account_backend` is the publish truth; `FANOPS_POSTER` is only the legacy bridge, and a stale `=dryrun` line is scraped on flip); an unknown `FANOPS_POSTER` resolves to dryrun (never a false LIVE banner). The "Publish by hand" tab stays the zero-infra fallback.
- Unfreeze learning (AUTOMATIC — no operator step): `learning_validated` reads `cutover.json metrics_confirmed`, and `track.pull_metrics` now AUTO-stamps it (`_auto_validate_metrics_shape`) the first time a real, **non-degraded** analyzed metric lands from a LIVE backend (the metric field-shape is thereby proven against `track._W` — exactly what the manual cutover reconciled by hand). dryrun never proves it (no real analytics); a degraded row (a primary weighted key absent or present-but-null) never stamps. This is a CORRECTNESS gate (don't learn on an unproven/mis-keyed shape), NOT an operator gate — it opens on real data. The Go-Live tab's **5 · Validate learning** step (the `cutover.py`/`cutover_postiz.py` throwaway-probe) still works as an optional EARLY manual shortcut (prove the shape before real posts accumulate) but is no longer required; it writes ONLY `cutover.json`, never the ledger, key never echoed.
- Insights culmination (analyzed reach biases the dims it doesn't yet reach — codemap `docs/CODEMAPS/insights-culmination.md`): IG performance is read from the **Meta Graph** (`track._default_list_posts`→`GraphInsightsClient`, the SOLE IG metric reader — `PostizMetricsClient`-for-IG is dead, the class stays only for non-IG use). Every varied clip axis now has reach feedback, each RANKED by raw Graph reach (not the engagement-skewed `lift_score`), each gated on `learning_validated` + `p4_unlocked` (≥8 attributed posts × ≥2 values, `validation_gate.py`), amplify/bias-only (C1 — never retire/cascade/track), own DEFAULT-OFF kill switch, VALIDATION-FROZEN (inert until proven even switch-ON), FAIL-SAFE (exception → logged once, ledger/brief byte-identical), and NO new auto-publish path (biases GENERATION + SCHEDULE only). Per dim: **length** (`clip_profile`) + **opening** (`first_frame_kind`) + **framing** (`top_bias`, stamped PER-ACCOUNT `resolve_top_bias` at the crosspost mint) all ride `apply_p4_dim_bias` via `_P4_DIMS` (`FANOPS_P4_DIM_BIAS`); **timing** (`publish_hour`/`publish_dow`, stamped in `cfg.operator_tz` — fails CLOSED to UTC — at the true publish transition in `post/run.py` + `reconcile.py`) rides `apply_timing_bias` → a control-file hour prior `crosspost.surface_time` consumes (`hour_hint`, window-clamped to `account_window`), a schedule-slot bias never a publish (`FANOPS_TIMING_BIAS`); **casting** (`account`×`clip_profile`) rides `casting_bias.casting_reach_prior` injected INLINE into the casting brief (`MomentCastingRequest.reach_prior`, the sibling of `_learned_account_signal` — NOT a cmd_run control-file), explore-guarded (ANNOTATES a proven cell, OMITS an under-exposed one, never removes an account from `personas` → no reach-monoculture/starvation; `FANOPS_CASTING_BIAS`). `apply_p4_dim_bias`/`apply_timing_bias` fire in `cmd_run` post-loop (casting's prior is recomputed at each brief). Built by the insights-rebuild plan Legs 2-3 (`.claude/plans/insights-culmination-MASTER.plan.md`); tests `test_culmination_coverage.py` + `test_casting_bias.py`.
- Per-account differentiation (M2+M3, 2026-06-23): each account gets its OWN clip LENGTH + FRAMING + on-screen hook + caption off one shared moment, reviewable at scale. `Account.clip_profile` (short/medium/long) + `Account.framing` (top/center) → `Config.resolve_clip_profile`/`resolve_top_bias` → an override account's `Render` is a real per-account CUT from the SOURCE at its own band+crop (`clip.render_account_cut`, fail-open), content-addressed `child_id("render", clip, hook[\x1fband:lo-hi][\x1fframe:x])` so same-hook-different-spec accounts never collide; `Render.is_account_cut` is the provenance truth. Review surfaces the spec per surface (length/cut/framing chips), bulk-approves by moment (`approve_clip`) or one-account-across-the-video (`approve_account`, scoped to `?batch=`), and has a `?compact=` dense video-less list. **`creative_variation` now DEFAULTS ON (M3d)** — per-account hooks/renders ship by default; `FANOPS_CREATIVE_VARIATION=0` restores the legacy single shared-clip+moment-hook path (the OFF code path + invariant `test_cv_off_mints_no_renders` retained, §7 firewall). The Review removed-hook RESTORE choice (`approve_with_hook`) is an OFF-mode flow — hidden while ON (per-surface hooks own the burn; the action refuses).
- Account-First Studio (SCHEMA_VERSION 8; the batch-target path is default-safe — an unbatched / `[]`-target source is render/post byte-identical. NB per-account **casting now DEFAULTS ON** — `FANOPS_ACCOUNT_CASTING=0` restores the legacy fan-to-all path): a **Batch** (`models.Batch`, `batches.create_batch`, `Ledger.batches`/`get_batch`) is a named, account-targeted ingest group — `target_accounts` (`[]`=all-active sentinel) is denormalized onto `Source.batch_id`→`Post.batch_id` and ENFORCED by a per-surface SKIP in `crosspost_clips` (the casting-OFF path; `@b` births no post + a `batch_target_skip`/`batch_target_summary` breadcrumb — the only record, since excluded surfaces never become posts). A zero-active-target batch is flagged advisory via `Batch.error_reason` (still mints) and surfaced on Home. The default-ON **casting** stage — now an LLM-driven per-account SELECTION (`casting.request_moment_casting`/`ingest_moment_casting`, the `moment_casting` agent gate; `cast_moments` is the kept token-overlap fallback, no longer wired) — picks each account its OWN moments via `Moment.affinities` (`FANOPS_ACCOUNT_CASTING` default ON + `FANOPS_CAST_PICK_BUDGET`, Go-Live **toggle**); the crosspost affinity gate then fans a cast moment ONLY to its accounts (an uncast moment falls through to fan-to-all). Both skips fail-open. Surfaces: status **Home** (`GET /` — accounts/connection + counts + clickable `len(batches)` entry + per-account post counts + zero-result-batch badge); **Review** batch-grouped + `?batch=` drill-in + target/state/created_at header + read-only cast chips + inline **re-burn** (writes `Post.variant_hook`+`media_urls` ONLY — `meta_captions['hook']` is dead post-hook-rewrite); **Schedule/Posted** paginated (`views.paginate` slice-then-regroup) + per-row batch label + `?batch=` filter + read-only per-batch "N posted · mean lift" rollup. Learning stays `learning_validated`-frozen — NO batch flag unfreezes it (rollup is a pure read over `Post.metrics[lift_score]`). Orphan flags `account_first`/`batch_studio` were never built (dropped). One cross-face proof: `tests/test_account_first_e2e.py` (slow UNIT, no marker).
- Hashtag lifecycle (personas-first, 2026-06-23): hashtags are now per-persona, evidence-backed, closed-loop — codemap `docs/CODEMAPS/hashtag-lifecycle.md`, doc `.claude/skills/fanops-hook-hashtag/SKILL.md` Part 3. A **first-class `Persona`** (`personas.py`, `00_control/personas.json`: `voice`/**`hashtag_corpus`** + lever fields; the per-account `tag_lean` was retired into the corpus, M3) is edited in the Studio **Personas** tab; `Account.persona_id` links it and the persona's voice/**corpus** HYDRATE the account at load (`_hydrate_from_personas`, fail-open, byte-identical when unlinked — every existing consumer unchanged). `migrate_from_accounts` lifts inline persona strings into records. The corpus feeds selection: `vet_hashtags(..., corpus=)` makes the corpus JOIN the vetted membership (a curated tag the frozen set/store doesn't know SURVIVES) + the PRIORITY pool (seeded first, leads the ≤4, ahead of the frozen rank); `caption.request_captions` carries each surface's corpus, `ingest_captions` applies it, `caption_prompt` shows it. Corpus tags come from: **live co-occurrence discovery** (`personas.discover_corpus`→`meta_graph.discover_candidates`/`harvest_cooccurring` — the Studio **Research corpus** button harvests the tags a category's currently-winning posts use via live `top_media` captions, finding tags NEVER named; IG has no trending-by-topic endpoint so this co-occurrence harvest is the Graph-native discovery; one `ig_hashtag_search` slot per seed, caption read free; FAIL-OPEN to the offline re-rank), **offline bootstrap re-rank** (`personas.research_corpus` — the no-creds fallback, store order, budget-free), **operator recommend** (`meta_graph.tag_metrics` — live IG reach for ONE tag, 30/7-day budget, token never echoed), or the frozen floor. Every source PROPOSES; the operator ACCEPTS into the corpus (curation gate) — discovery NEVER auto-writes a tag into a caption. `fanops hashtags discover` (`cmd_hashtags_discover`) is the periodic per-persona REPORT (launchd/cron), READ-ONLY w.r.t. the menu. **`VETTED` is now the cold-start FLOOR, not canonical.** (Global auto-absorption of unvetted discoveries was deliberately NOT built — an engagement floor admits generic spam + bypasses the operator gate.) Graph-reach judging (2026-06-27): a hashtag's worth is its LIVE Meta Graph reach, NEVER a post that used it — post insights attribute to the hook/clip/account-in-stitch, never the hashtag (invariant pinned by `tests/test_hashtag_attribution_severance.py`). `fanops_hashtags.refresh_store(cfg)` harvests co-occurring candidates from the niche seeds (persona corpora + intake.genre), measures their Graph reach within the 30/7-day budget, ranks by reach, and writes the store — NO ledger, NO doctor gate (`FANOPS_HASHTAG_TRENDS` **DEFAULTS ON**, fail-open to the frozen floor without a Meta token); `refresh_store_if_due` re-runs it on a 12h throttle inside `fanops run` (each daemon tick). The store persists `{tags, reach}`; provenance source `graph-reach`; the Personas tab renders the corpus reach-ranked + ★ most-active + each tag's live Graph reach. e2e: `tests/test_hashtag_lifecycle_e2e.py`.
- Dynamic reframe (universal, content- AND aspect-adaptive, 2026-06-28, DEFAULTS ON — `FANOPS_SMART_FRAMING=0` restores the blind centered crop, byte-identical): a STABLE, LOCKED-OFF-camera reframe for talking / music / silent footage in landscape / portrait / square / already-9:16 sources, to any `_TARGETS` aspect. **Each shot is ONE fixed, correctly-sized crop held perfectly still, hard-cutting between speakers — a locked-off virtual camera, how real clippers cut it.** (A per-frame chase reads as a jittery hand-held cam — it tracks every detection wobble and the zoom "breathes"; a per-window single crop can't size two different speakers; both were tried and rejected. The static-per-shot crop is jitter-free BY CONSTRUCTION — the ffmpeg crop is constant within a segment, no `t`-expression.) Pipeline: (1) `framing.detect_window` runs ONE cheap grid pass (`keyframes.extract_frames_grid`) detecting every face's `(cx,cy,fh,ey)` via vendored YuNet, cached `<src>.detect.json`; (2) `framing.classify_window` → `multi-speaker-talk | single-speaker-talk | music | silent | no-people`; (3) `clip._resolve_framing` → `(focus, track, content_type)`: active-speaker `framing.speaker_track` (mouth-motion @ `_ASD_FPS` 9fps + `_ASD_HOLD_S` 0.35s hysteresis → `(t0,t1,fx,fy,fh,ey)` segments, fh as p75 = clearest full-face not occlusion-dragged median; `_merge_brief_segments` absorbs shots < `_ASD_MIN_SEG_S` 1.5s so a brief interjection never triggers a cut-away-and-back) for real 2-shots; static `framing.subject_focus` for single/music/silent; `motion_saliency` for no-face; centered otherwise. **RENDER (`render_reframed` fail-open ladder): a real 2-shot → `ffmpeg_segments_cmd` (each speaker's segment its OWN static crop, joined by the concat filter in one pass, sample-accurate audio + subtitle burn); else → single-pass `ffmpeg_clip_cmd` (one static subject-lock or centered crop).** Crop sizing (`_focus_crop` / `_segment_chain` via `_crop_box`/`_zoom_h`): zoom so the face fills `_FACE_FRAC_TALK` (0.42) / `_FACE_FRAC_MUSIC` (0.26), eyes at `_EYELINE_FRAC` (0.40). **Zoom is FACE-SIZE-ADAPTIVE** (`_adaptive_zoom_max`): a near/camera-facing subject punches in (`_ZOOM_MAX` 1.6 single / `_ZOOM_MAX_TRACK` 1.7 segment), a FAR/small one (`fh < _SMALL_FACE_FRAC` 0.18 — typically profile + mic-occluded) is held WIDE (`_ZOOM_MAX_FAR` 1.25) showing context, never a tight crop of the foreground mic (a mic physically in front of a face can't be cropped out — frame wide instead). OpenCV is the OPTIONAL `[framing]` extra (detection only — the render is ffmpeg), lazy + **FAIL-OPEN at every step** → centered crop, so default-on is never worse than before. `_render_fingerprint` hashes focus/track + `ct`/`geom` (`_REFRAME_GEOM_V` 4) ONLY when a zoom applies (centered clips keep their hash → no needless re-render). Acceptance is the VISUAL on rendered frames (the YuNet face-box metric under-reads on caps/profiles/downturned heads — a still can't show jitter or size-drift either way; check adjacent frames have a fixed crop). Tests: `tests/test_smart_framing.py`, `tests/test_keyframes.py`.

## Constraints

- NEVER mass-reformat: no `black`, no `ruff format`. The compact one-liner house style
  (E701/E702/E401/E501 ignored) is deliberate — rationale in pyproject.toml comments.
- The global 60s pytest timeout is a deadlock guardrail (ledger flock). A hanging test
  is the bug; don't raise the timeout to make it pass.
- The `fanops` CLI has live verbs that hit external services (Postiz publish, Meta Graph
  metrics). Don't run it speculatively; tests and read-only verbs only unless the operator asks.
- `.claude/workflows/*.js` are tracked, load-bearing build workflows — never delete.
