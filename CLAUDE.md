## Commands

- Install: `pip install -e '.[dev]'` ‚Äî extras: `[studio]` (Flask cockpit, imported lazily), `[transcribe]` (whisper CLI), `[compose]` (MoviePy produced-clip compositing ‚Äî `fanops compose`, lazy + fail-open)
- Fast unit suite (CI `unit` job): `python -m pytest -q -m "not integration"`
- Integration suite (CI `e2e` job): `python -m pytest -q -m integration -rs` ‚Äî needs real ffmpeg/ffprobe/whisper/espeak on PATH; skips locally when absent; CI sets `FANOPS_REQUIRE_E2E=1` so a skip fails
- Lint: `ruff check .` (pyflakes F + pycodestyle E only)
- Studio dev server: `fanops studio` (localhost:8787; requires `[studio]` extra)
- Browser ingestion (no Finder): the Studio **Run** tab **Upload video** form streams raw video into `01_inbox/` (validated: video ext, traversal-safe `secure_filename`, inbox-bound resolve, atomic `.uploadpart`‚Üí`os.replace`; 2 GiB `MAX_CONTENT_LENGTH` cap) ‚Üí click **Ingest inbox** to catalogue it. `actions.save_uploads` owns the contract; an oversize body re-renders the panel at HTTP 200 ("too large") since htmx 2.x drops non-2xx swaps.
- Approval lifecycle (no auto-publish): a crossposted post is BORN `PostState.awaiting_approval` ‚Äî `publish_due`/`publish_now` iterate only `queued`, so NOTHING publishes (even on a live backend / daemon) until the operator approves it. The Studio **Review** tab is the approve worklist (per-surface checkboxes ‚Üí batch **Approve selected** / **Reject selected**; `Ledger.approve_post` promotes awaiting‚Üíqueued). Per-post-per-surface scheduling is first-class (P1): approving an UNTIMED / stale-past post no longer bumps it to **now** (which silently published it on the next `publish_due`) ‚Äî it lands a deterministic **strictly-future suggestion** (`views.suggest_time` = `crosspost.surface_time` at `index=0`, never a 40-min stagger), so it sits in `queued` for the lead window instead of firing instantly; a still-future operator-set time is preserved verbatim. Every editable Review surface AND Schedule row offers **Use suggested** (one-click accept the suggestion) and **Clear time** (`actions.clear_time` ‚Äî on a queued post it FIRST un-approves back to Review, THEN clears, so a post is never left `queued`-and-timeless). The **Schedule** tab is the approved-posts bucket cockpit (shows each post's Postiz integration; per-row Move / Use suggested / Clear time / Publish now / Send back to Review; **Reschedule all** re-spreads via `crosspost.surface_time` ‚Äî the 40-min auto-stagger is reachable ONLY here, never imposed on an individual post). The **Posted** tab is the all-time shipped library (live URL + lift) + **Post again** (`actions.repost_post` spawns a fresh awaiting_approval repost of the same clip with an epoch-suffixed id ‚Äî honors fan-accounts-repost-freely, NOT a supersede). `queued` now means "approved + scheduled"; `rejected` is an operator discard. Mirrors the M3/M4 stitch approve/release gate.
- Onboard + go live (no env vars / CLI / JSON): the Studio **Go Live** tab connects Postiz, **adds an account** (handle + platforms), maps **each (handle √ó platform) channel** to its own Postiz integration id (`accounts.json` `integrations` is per-platform ‚Äî a handle's IG and TikTok are different integrations; a legacy single `account_id` stays the fallback), and flips dryrun‚Üîlive behind a confirm (dual-writes `.env` + `os.environ`; the API key is write-only ‚Äî never rendered). `go_live` is the only setter of `FANOPS_POSTER=postiz`, gated on readiness + confirm; an unknown `FANOPS_POSTER` resolves to dryrun (never a false LIVE banner). The "Publish by hand" tab stays the zero-infra fallback.
- Unfreeze learning (AUTOMATIC ‚Äî no operator step): `learning_validated` reads `cutover.json metrics_confirmed`, and `track.pull_metrics` now AUTO-stamps it (`_auto_validate_metrics_shape`) the first time a real, **non-degraded** analyzed metric lands from a LIVE backend (the metric field-shape is thereby proven against `track._W` ‚Äî exactly what the manual cutover reconciled by hand). dryrun never proves it (no real analytics); a degraded row (a primary weighted key absent or present-but-null) never stamps. This is a CORRECTNESS gate (don't learn on an unproven/mis-keyed shape), NOT an operator gate ‚Äî it opens on real data. The Go-Live tab's **5 ¬∑ Validate learning** step (the `cutover.py`/`cutover_postiz.py` throwaway-probe) still works as an optional EARLY manual shortcut (prove the shape before real posts accumulate) but is no longer required; it writes ONLY `cutover.json`, never the ledger, key never echoed.
- Insights culmination (analyzed reach biases the dims it doesn't yet reach ‚Äî codemap `docs/CODEMAPS/insights-culmination.md`): IG performance is read from the **Meta Graph** (`track._default_list_posts`‚Üí`GraphInsightsClient`, the SOLE IG metric reader ‚Äî `PostizMetricsClient`-for-IG is dead, the class stays only for non-IG use). Every varied clip axis now has reach feedback, each RANKED by raw Graph reach (not the engagement-skewed `lift_score`), each gated on `learning_validated` + `p4_unlocked` (‚â•8 attributed posts √ó ‚â•2 values, `validation_gate.py`), amplify/bias-only (C1 ‚Äî never retire/cascade/track), own DEFAULT-OFF kill switch, VALIDATION-FROZEN (inert until proven even switch-ON), FAIL-SAFE (exception ‚Üí logged once, ledger/brief byte-identical), and NO new auto-publish path (biases GENERATION + SCHEDULE only). Per dim: **length** (`clip_profile`) + **opening** (`first_frame_kind`) + **framing** (`top_bias`, stamped PER-ACCOUNT `resolve_top_bias` at the crosspost mint) all ride `apply_p4_dim_bias` via `_P4_DIMS` (`FANOPS_P4_DIM_BIAS`); **timing** (`publish_hour`/`publish_dow`, stamped in `cfg.operator_tz` ‚Äî fails CLOSED to UTC ‚Äî at the true publish transition in `post/run.py` + `reconcile.py`) rides `apply_timing_bias` ‚Üí a control-file hour prior `crosspost.surface_time` consumes (`hour_hint`, window-clamped to `account_window`), a schedule-slot bias never a publish (`FANOPS_TIMING_BIAS`); **casting** (`account`√ó`clip_profile`) rides `casting_bias.casting_reach_prior` injected INLINE into the casting brief (`MomentCastingRequest.reach_prior`, the sibling of `_learned_account_signal` ‚Äî NOT a cmd_run control-file), explore-guarded (ANNOTATES a proven cell, OMITS an under-exposed one, never removes an account from `personas` ‚Üí no reach-monoculture/starvation; `FANOPS_CASTING_BIAS`). `apply_p4_dim_bias`/`apply_timing_bias` fire in `cmd_run` post-loop (casting's prior is recomputed at each brief). Built by the insights-rebuild plan Legs 2-3 (`.claude/plans/insights-culmination-MASTER.plan.md`); tests `test_culmination_coverage.py` + `test_casting_bias.py`.
- Per-account differentiation (M2+M3, 2026-06-23): each account gets its OWN clip LENGTH + FRAMING + on-screen hook + caption off one shared moment, reviewable at scale. `Account.clip_profile` (short/medium/long) + `Account.framing` (top/center) ‚Üí `Config.resolve_clip_profile`/`resolve_top_bias` ‚Üí an override account's `Render` is a real per-account CUT from the SOURCE at its own band+crop (`clip.render_account_cut`, fail-open), content-addressed `child_id("render", clip, hook[\x1fband:lo-hi][\x1fframe:x])` so same-hook-different-spec accounts never collide; `Render.is_account_cut` is the provenance truth. - Dynamic reframe (universal, content- AND aspect-adaptive, 2026-06-28, DEFAULTS ON ‚Äî `FANOPS_SMART_FRAMING=0` restores the blind centered crop, byte-identical): a STABLE, LOCKED-OFF-camera reframe for talking / music / silent footage in landscape / portrait / square / already-9:16 sources, to any `_TARGETS` aspect. **Each shot is ONE fixed, correctly-sized crop held perfectly still, hard-cutting between speakers ‚Äî a locked-off virtual camera, how real clippers cut it.** (A per-frame chase reads as a jittery hand-held cam ‚Äî it tracks every detection wobble and the zoom "breathes"; a per-window single crop can't size two different speakers; both were tried and rejected. The static-per-shot crop is jitter-free BY CONSTRUCTION ‚Äî the ffmpeg crop is constant within a segment, no `t`-expression.) Pipeline: (1) `framing.detect_window` runs ONE cheap grid pass (`keyframes.extract_frames_grid`) detecting every face's `(cx,cy,fh,ey)` via vendored YuNet, cached `<src>.detect.json`; (2) `framing.classify_window` ‚Üí `multi-speaker-talk | single-speaker-talk | music | silent | no-people`; (3) `clip._resolve_framing` ‚Üí `(focus, track, content_type)`: active-speaker `framing.speaker_track` (mouth-motion @ `_ASD_FPS` 9fps + `_ASD_HOLD_S` 0.35s hysteresis ‚Üí `(t0,t1,fx,fy,fh,ey)` segments, fh as p75 = clearest full-face not occlusion-dragged median; `_merge_brief_segments` absorbs shots < `_ASD_MIN_SEG_S` 1.5s so a brief interjection never triggers a cut-away-and-back) for real 2-shots; static `framing.subject_focus` for single/music/silent; `motion_saliency` for no-face; centered otherwise. **RENDER (`render_reframed` fail-open ladder): a real 2-shot ‚Üí `ffmpeg_segments_cmd` (each speaker's segment its OWN static crop, joined by the concat filter in one pass, sample-accurate audio + subtitle burn); else ‚Üí single-pass `ffmpeg_clip_cmd` (one static subject-lock or centered crop).** Crop sizing (`_focus_crop` / `_segment_chain` via `_crop_box`/`_zoom_h`): zoom so the face fills `_FACE_FRAC_TALK` (0.42) / `_FACE_FRAC_MUSIC` (0.26), eyes at `_EYELINE_FRAC` (0.40). **Zoom is FACE-SIZE-ADAPTIVE** (`_adaptive_zoom_max`): a near/camera-facing subject punches in (`_ZOOM_MAX` 1.6 single / `_ZOOM_MAX_TRACK` 1.7 segment), a FAR/small one (`fh < _SMALL_FACE_FRAC` 0.18 ‚Äî typically profile + mic-occluded) is held WIDE (`_ZOOM_MAX_FAR` 1.25) showing context, never a tight crop of the foreground mic (a mic physically in front of a face can't be cropped out ‚Äî frame wide instead). OpenCV is the OPTIONAL `[framing]` extra (detection only ‚Äî the render is ffmpeg), lazy + **FAIL-OPEN at every step** ‚Üí centered crop, so default-on is never worse than before. `_render_fingerprint` hashes focus/track + `ct`/`geom` (`_REFRAME_GEOM_V` 4) ONLY when a zoom applies (centered clips keep their hash ‚Üí no needless re-render). Acceptance is the VISUAL on rendered frames (the YuNet face-box metric under-reads on caps/profiles/downturned heads ‚Äî a still can't show jitter or size-drift either way; check adjacent frames have a fixed crop). Tests: `tests/test_smart_framing.py`, `tests/test_keyframes.py`.

## Constraints

- NEVER mass-reformat: no `black`, no `ruff format`. The compact one-liner house style
  (E701/E702/E401/E501 ignored) is deliberate ‚Äî rationale in pyproject.toml comments.
- The global 60s pytest timeout is a deadlock guardrail (ledger flock). A hanging test
  is the bug; don't raise the timeout to make it pass.
- The `fanops` CLI has live verbs that hit external services (Postiz publish, Meta Graph
  metrics). Don't run it speculatively; tests and read-only verbs only unless the operator asks.
- `.claude/workflows/*.js` are tracked, load-bearing build workflows ‚Äî never delete.
