- Approval lifecycle (no auto-publish): a crossposted post is BORN `PostState.awaiting_approval` ŌĆö `publish_due`/`publish_now` iterate only `queued`, so NOTHING publishes (even on a live backend / daemon) until the operator approves it. The Studio **Review** tab is the approve worklist (per-surface checkboxes ŌåÆ batch **Approve selected** / **Reject selected**; `Ledger.approve_post` promotes awaitingŌåÆqueued). Per-post-per-surface scheduling is first-class (P1): approving an UNTIMED / stale-past post no longer bumps it to **now** (which silently published it on the next `publish_due`) ŌĆö it lands a deterministic **strictly-future suggestion** (`views.suggest_time` = `crosspost.surface_time` at `index=0`, never a 40-min stagger), so it sits in `queued` for the lead window instead of firing instantly; a still-future operator-set time is preserved verbatim. Every editable Review surface AND Schedule row offers **Use suggested** (one-click accept the suggestion) and **Clear time** (`actions.clear_time` ŌĆö on a queued post it FIRST un-approves back to Review, THEN clears, so a post is never left `queued`-and-timeless). The **Schedule** tab is the approved-posts bucket cockpit (shows each post's Postiz integration; per-row Move / Use suggested / Clear time / Publish now / Send back to Review; **Reschedule all** re-spreads via `crosspost.surface_time` ŌĆö the 40-min auto-stagger is reachable ONLY here, never imposed on an individual post). The **Posted** tab is the all-time shipped library (live URL + lift) + **Post again** (`actions.repost_post` spawns a fresh awaiting_approval repost of the same clip with an epoch-suffixed id ŌĆö honors fan-accounts-repost-freely, NOT a supersede). `queued` now means "approved + scheduled"; `rejected` is an operator discard. Mirrors the M3/M4 stitch approve/release gate.
- Onboard + go live (no env vars / CLI / JSON): the Studio **Go Live** tab connects Postiz, **adds an account** (handle + platforms), maps **each (handle ├Ś platform) channel** to its own Postiz integration id (`accounts.json` `integrations` is per-platform ŌĆö a handle's IG and TikTok are different integrations; a legacy single `account_id` stays the fallback), and flips dryrunŌåölive behind a confirm (dual-writes `.env` + `os.environ`; the API key is write-only ŌĆö never rendered). `go_live` is the only setter of `FANOPS_POSTER=postiz`, gated on readiness + confirm; an unknown `FANOPS_POSTER` resolves to dryrun (never a false LIVE banner). The "Publish by hand" tab stays the zero-infra fallback.
- Unfreeze learning (AUTOMATIC ŌĆö no operator step): `learning_validated` reads `cutover.json metrics_confirmed`, and `track.pull_metrics` now AUTO-stamps it (`_auto_validate_metrics_shape`) the first time a real, **non-degraded** analyzed metric lands from a LIVE backend (the metric field-shape is thereby proven against `track._W` ŌĆö exactly what the manual cutover reconciled by hand). dryrun never proves it (no real analytics); a degraded row (a primary weighted key absent or present-but-null) never stamps. This is a CORRECTNESS gate (don't learn on an unproven/mis-keyed shape), NOT an operator gate ŌĆö it opens on real data. The Go-Live tab's **5 ┬Ę Validate learning** step (the `cutover.py`/`cutover_postiz.py` throwaway-probe) still works as an optional EARLY manual shortcut (prove the shape before real posts accumulate) but is no longer required; it writes ONLY `cutover.json`, never the ledger, key never echoed.
- Per-account differentiation (M2+M3, 2026-06-23): each account gets its OWN clip LENGTH + FRAMING + on-screen hook + caption off one shared moment, reviewable at scale. `Account.clip_profile` (short/medium/long) + `Account.framing` (top/center) ŌåÆ `Config.resolve_clip_profile`/`resolve_top_bias` ŌåÆ an override account's `Render` is a real per-account CUT from the SOURCE at its own band+crop (`clip.render_account_cut`, fail-open), content-addressed `child_id("render", clip, hook[\x1fband:lo-hi][\x1fframe:x])` so same-hook-different-spec accounts never collide; `Render.is_account_cut` is the provenance truth. ## Constraints

- NEVER mass-reformat: no `black`, no `ruff format`. The compact one-liner house style
  (E701/E702/E401/E501 ignored) is deliberate ŌĆö rationale in pyproject.toml comments.
- The global 60s pytest timeout is a deadlock guardrail (ledger flock). A hanging test
  is the bug; don't raise the timeout to make it pass.
- The `fanops` CLI has live verbs that hit external services (Blotato). Don't run it
  speculatively; tests and read-only verbs only unless the operator asks.
- `.claude/workflows/*.js` are tracked, load-bearing build workflows ŌĆö never delete.
