## Pipeline overview (raw file -> catalogued source, end to end)

Three entry paths feed the same catalogue spine:

1. **Browser upload / manual drop** — a video lands in `01_inbox/` (Studio's upload form writes it there atomically; see CLAUDE.md). Nothing happens until `ingest_drops` runs (CLI `ingest` or Studio "Ingest inbox").
2. **URL pull** — `download_url` shells `yt-dlp` into an isolated staging dir `01_inbox/.pull/`, returns the set of newly-produced file paths (before/after snapshot diff), then `ingest_drops(origin="url", inbox=stage, origin_paths=produced)` catalogues only those files. `download_source` is the non-split convenience wrapper (download + ingest in one call, held for direct callers/tests); the CLI's real `pull` command splits the two so the network call never runs inside the ledger lock.
3. **Local folder scan / review-first intake** — `discover.discover()` cheaply scans arbitrary filesystem roots (`scan_local`), writes a thumbnail + manifest entry per new candidate into `00_review/` for the operator to eyeball in Finder. The operator moves approved thumbnails into `00_review/approved/`; `discover.intake()` then copies the corresponding originals into `01_inbox/`, at which point they become ordinary drops for `ingest_drops` to catalogue on the next pass.

`ingest_drops` (the single spine, called via `_catalogue_file` per file) does, per file in the inbox:
- sweeps partial upload/download temps (`_sweep_partials`)
- re-probes any source previously stuck at 0×0 (`_reprobe_degraded`)
- filters symlinks, non-media extensions, and already-archived files
- filters PII/legal-named files (`is_excluded`) → archives without cataloguing
- filters audio-only files (`has_video_stream` False) → archives without cataloguing
- computes `sha256_of` the file for content-addressed dedup (`led.already_seen`)
- if new: copies into `cfg.sources/<id><ext>`, probes width/height/duration (`probe_dimensions`), mints a `Source` (state `catalogued`), threads a batch_id (auto-minted drop-batch if none supplied)
- archives the disposed inbox file into `01_inbox/.ingested/` (ING-1: keeps the inbox steady-state empty so no re-hash)

Downstream of cataloguing, `transcribe.transcribe_source` (called from `pipeline._stage_source_to_moments` / `produce._produce_one`) turns a catalogued Source into a `transcribed` one: optional Demucs vocal isolation (`vocals.isolate_vocals`) feeds a cleaner signal into Whisper/faster-whisper, producing `Source.transcript` + `Source.language`.

`adjust.py` and `intro_match.py` sit downstream of publishing, not upstream of ingest — they're post-hoc learning/pairing stages that happen to live in this cluster's file list; they don't touch raw source acquisition.

`bands.py`, `frames.py`, `audio_energy.py`, `vocals.py` are pure-analysis/support modules consumed by the clip-selection stage (in `clip.py` / `signals.py`, outside this cluster) to decide clip length and framing.

## Per-file breakdown

### ingest.py — the native-ingest spine (drop/url/local-scan → catalogued Source)

- `MEDIA_EXT` (module const) — recognized video/image/audio extensions accepted for ingest.
- `_PII` (module const, compiled regex) — case-insensitive filename pattern matching passport/id/visa/license/agreement/contract/invoice/nda/tax/bank/ssn/emirates-id/national-id.
- `is_excluded(name: str) -> bool` — True iff `name` matches `_PII`. No side effects. Called by `ingest_drops`, `scan_local`, `studio.actions_run.run_ingest_thirdparty`.
- `_archive_dir(inbox: Path) -> Path` — returns `inbox/.ingested`. Pure path builder.
- `_pull_stage(cfg) -> Path` — returns/creates `cfg.inbox/.pull` (mkdir side effect). Used to isolate a URL pull's yt-dlp output from manual drops.
- `_archive_inbox_file(inbox, f, cfg) -> None` — moves a disposed inbox file into `.ingested/` via `os.replace` (atomic same-fs move); disambiguates name collisions with an mtime-ns suffix. **Side effect**: filesystem write/move. Fail-open: an `OSError` (perms/cross-device) is caught and logged (`get_logger`), file left in place for retry — never raised.
- `_sweep_partials(inbox, cfg) -> None` — deletes leaked `*.uploadpart`/`*.part` temp files before scanning. **Side effect**: `f.unlink()`. Fail-open: `OSError` caught, logged, not raised.
- `_reprobe_degraded(led, cfg) -> None` — iterates `led.sources`, re-runs `probe_dimensions` on any Source with `degraded_reason == "probe_failed"` whose file still exists; on success, mutates the ledger's in-memory Source (via `model_copy`) clearing the degraded flag. **Side effect**: mutates `led.sources` dict in place, shells ffprobe (via `probe_dimensions`), logs on success.
- `sha256_of(path: Path) -> str` — streams the file in 1 MiB chunks through `hashlib.sha256`; returns hex digest. Pure I/O read, no mutation. Called by `discover.discover`, `_catalogue_file`.
- `_FFPROBE_TIMEOUT = 30.0`, `_YTDLP_TIMEOUT = 600.0` — module constants (hard subprocess bounds).
- `_run_ffprobe(args: list[str]) -> subprocess.CompletedProcess` — shells `ffprobe`. **Shells out to ffprobe.** Absent binary (`FileNotFoundError`/`OSError`) → raises `ToolchainMissingError` (typed, propagates to `cli.main` for a clean exit 2). A hang past 30s → caught `TimeoutExpired`, converted to a synthetic `CompletedProcess(returncode=124, stdout="")` — deliberately NOT re-raised, so a single corrupt file/stuck mount fails soft per-file rather than aborting the whole ingest transaction.
- `probe_dimensions(path: Path) -> tuple[int,int,float]` — runs ffprobe (via `_run_ffprobe`) to get width/height/duration; parses stdout tokens; returns `(0,0,0.0)` on any `IndexError`/`ValueError` (malformed/empty output). Shells ffprobe indirectly. Called by `clip._probe_duration`, `discover.candidate_meta`, `_catalogue_file`, `_reprobe_degraded`, `signals.detect_signals`.
- `has_video_stream(path: Path) -> bool` — runs ffprobe `-select_streams v:0 -show_entries stream=codec_type`; token-matches `"video"` in the comma/space-split output (guards against a trailing-comma quirk on some HEVC `.mov` muxings that would otherwise misclassify a real video as audio-only). Shells ffprobe indirectly. Called by `ingest_drops`, `studio.actions_run.save_uploads`.
- `_catalogue_file(led, cfg, f, *, origin, now_iso, origin_kind="native", batch_id=None) -> bool` — the atomic per-file catalogue operation: hashes the file, checks `led.already_seen`; if a dupe, logs conflicts (origin_kind mismatch, batch_id mismatch, source_origin mismatch) as visible breadcrumbs (never silently resolved) and returns True (safe to archive). If new: mints `sid = make_id("src", digest)`, copies file to `cfg.sources/<sid><ext>` via `shutil.copy2` (**disk write**), probes dimensions, flags `degraded_reason="probe_failed"` if width/height are 0, calls `led.add_source(...)` (**mutates ledger**). Returns False only on a copy `OSError` (ENOSPC/perms) — logs and leaves the file for retry, does NOT catalogue it.
- `_inbox_media(inbox: Path) -> set[Path]` — resolved paths of all media files currently under `inbox` (recursive glob, filters symlinks + MEDIA_EXT). Pure read, used for before/after diffing in `download_url`.
- `ingest_drops(led, cfg, *, origin="drop", origin_kind="native", inbox=None, batch_id=None, origin_paths=None) -> tuple[Ledger, IngestCounts]` — **the main ingest entry point.** Ensures `cfg.sources`/inbox dirs exist (mkdir side effect); sweeps partials; re-probes degraded sources; single clock read (`now_iso`); iterates `sorted(box.rglob("*"))` in the inbox, skipping symlinks/non-media/`.gitkeep`/already-archived files; routes PII-excluded and audio-only files straight to archive with a counted+logged skip; for genuine candidates, resolves `file_origin` per-file (a url-pull only tags files that were actually produced by that pull, via `origin_paths`); lazily mints a day-stable drop-batch on first real catalogue via `fanops.batches.resolve_or_mint_drop_batch` (deferred import); calls `_catalogue_file`; archives disposed files. Returns `(led, IngestCounts)`. **Side effects**: mkdir, file moves/copies, ledger mutation, logging, shells ffprobe transitively. Called by `cli._dispatch`, `download_source`, `pipeline.advance`, `studio.actions_run.run_ingest`/`run_ingest_thirdparty`/`run_pull`.
- `download_url(cfg, url: str) -> set[Path]` — **shells `yt-dlp`** into the isolated `.pull` staging dir with a before/after path-set diff to determine exactly what this call produced (robust to yt-dlp's merge/rename behavior, version-independent vs. parsing stdout). Absent binary → `ToolchainMissingError`. Nonzero return code (dead/geoblocked URL) → raises typed `DownloadError` with the stderr tail (prevents the prior silent-failure mode of reporting "pulled → 0 sources" as success). A hang past 600s → `TimeoutExpired` propagates uncaught by design (bubbles to `cli.main`'s guard). Holds NO ledger lock — deliberate, so a slow network call never blocks the flock. Called by `cli._dispatch`, `download_source`, `studio.actions_run.run_pull`.
- `download_source(led, cfg, url) -> Ledger` — convenience wrapper: `download_url` then `ingest_drops(origin="url", inbox=_pull_stage(cfg), origin_paths=produced)`. Kept for direct callers/tests; the real CLI `pull` path splits these two calls itself (network call outside the transaction, ingest inside a tight one). No repo callers besides tests per call graph (`called_by_in_repo: []`).
- `scan_local(roots: list[Path]) -> list[str]` — recursively walks each root, skips symlinks, filters by `MEDIA_EXT` and `is_excluded`; returns sorted string paths. Pure read, no probing/hashing (cheap). Called by `discover.discover`.

**Class `IngestCounts`** (`@dataclass`) — this-pass tally: `added`, `deduped`, `excluded`, `skipped` (all default 0). Pure data holder, no methods beyond dataclass defaults.

### discover.py — pre-ingest folder review (cheap scan → operator-reviewed manifest → intake into inbox)

- `_load_json(path, default)` — reads+parses a control-file JSON; on `JSONDecodeError`/`OSError` raises typed `ControlFileError` (never a raw traceback). Returns `default` if the file doesn't exist. Read-only.
- `_write_json_atomic(path, obj) -> None` — writes JSON to a `.tmp` sibling then `os.replace`s it into place (atomic same-fs swap, asserted via `assert tmp.parent == path.parent`). **Side effect**: disk write. No lock held (module docstring: single-operator assumption — concurrent `discover`/`intake` runs on the same root can lost-update these files by design; each individual write is still crash-safe).
- `candidate_meta(path: Path) -> dict` — `os.stat` for bytes/mtime; calls `probe_dimensions` (shells ffprobe) for width/height/duration, fail-soft: `ToolchainMissingError` → logs a warning breadcrumb, dims stay `None`; any other `Exception` → logs debug, dims stay `None`. Never raises. Called by `discover.discover`.
- `_THUMB_TIMEOUT = 60.0` (module const).
- `make_thumbnail(path, out_jpg, *, at_seconds=1.0) -> bool` — **shells `ffmpeg`** to grab one 320px-wide frame at 1s; on failure/absence/timeout, retries once at `t=0` (handles sub-1s clips); returns `False` on any failure (fail-open — the candidate is still listed without a thumb, never raises). **Side effect**: writes a `.jpg` file, creates parent dirs. Called by `discover.discover`, `studio.app.create_app`.
- `discover(cfg, roots: list[Path]) -> dict` — the main scan entry: ensures `cfg.review` + `cfg.review/approved` exist (mkdir), loads a `Ledger` (`Ledger.load`), loads/updates `manifest.json`; for each candidate from `scan_local`, computes sha256, skips if already a known Source (`led.already_seen`) or already in the manifest; otherwise gets `candidate_meta`, makes a thumbnail, adds a manifest entry keyed by `digest[:16]`. Writes the manifest atomically at the end. Returns `{found, new, skipped}`. **Side effects**: mkdir, ledger load (read-only), thumbnail files, manifest JSON write. No repo callers found in call graph (`called_by_in_repo: []` — likely invoked only from the CLI dispatch table, not tracked as a "call" in the AST graph, or from `fanops-discover` skill/CLI verb).
- `intake(cfg) -> dict` — sweeps `cfg.review/approved/*.jpg`; for each approved thumbnail not already in `intaken.json`, looks up its manifest entry, resolves the original `source_path`, and (if it exists and isn't a symlink) copies it into `cfg.inbox` via `shutil.copy2`. Missing/symlinked/manifest-less entries are counted as `missing`, never raised. Writes `intaken.json` atomically at the end. Returns `{approved, intaken, missing}`. **Side effects**: file copy into `01_inbox/`, `intaken.json` write. No repo callers found in call graph — invoked via CLI dispatch, same as `discover`.

### adjust.py — post-publish learning stage (winner/loser classification, amplify, retire) — NOT part of raw ingest, downstream of publish/analyze

- `MAX_AMPLIFY_PER_SOURCE = 3` (module const) — shared cap between `amplify()`'s default and `variant_amplify.amplify_candidates`'s pre-check.
- `classify_outcomes(led, *, winner_pct=0.3, retire_pct=0.2, lift_floor=20.0, per_surface=False) -> dict` — filters posts to `state is analyzed and LIFT_SCORE in metrics` (excludes failed posts, which carry no lift_score — FIX F22); ranks by lift_score descending. Winners: either global top `winner_pct` slice, or (if `per_surface=True`) per-`(account, platform)` bucketed top slice, unioned. Losers: always global bottom `retire_pct` slice, further filtered to exclude anything already a winner and below `lift_floor`. Pure function over `led` (read-only on posts, no mutation). Returns `{"winners": [...], "losers": [...]}` (post ids). Called by `cli._learn_pass`, `cli.cmd_adjust`.
- `amplify(led, cfg, winner_post_ids, *, max_amplify_per_source=MAX_AMPLIFY_PER_SOURCE, extra_guidance="") -> Ledger` — for each winner post, walks post→clip→moment→source; skips if the source has already hit the per-source amplify cap (`src.meta.get("amplify_count", 0)`); builds an LLM guidance string referencing the winning moment's transcript excerpt and lift; constructs a `MomentRequest` payload and calls `agentstep.write_request(cfg, kind="moments", key=src.id, payload=...)` — **this invalidates any stale prior response** (deliberate, so `ingest_moments` re-answers fresh). **Side effects**: mutates `src.meta["amplify_count"]`, calls `led.set_source_state(src.id, moments_requested)`, writes a request control file (via `write_request`, in `agentstep.py`, outside this cluster). Called by `cli._learn_pass`, `cli.cmd_adjust`, `p4_dim_bias.apply_p4_dim_bias`, `variant_amplify.apply_variant_amplify`.
- `retire(led, loser_post_ids) -> Ledger` — for each loser post, calls `led.retire_clip(post.parent_id)`; if no live sibling clip remains for that moment, also sets the moment to `MomentState.retired` (else a later render pass would resurrect it — the render guard checks moment state). **Side effect**: mutates `led` (clip/moment state). Called by `cli._learn_pass`, `cli.cmd_adjust`.

### bands.py — clip-length band constants and profile resolution (pure config/lookup, no ingest)

- **Class `Band(NamedTuple)`** — `lo: float`, `hi: float`; property `span` = midpoint `(lo+hi)/2`.
- `TALK = Band(12.0, 22.0)`, `SONG = Band(18.0, 35.0)`, `SHORT = Band(8.0, 15.0)`, `MEDIUM = Band(16.0, 26.0)`, `LONG = Band(28.0, 45.0)` — module-level constant instances.
- `_PROFILES` (dict), `PROFILE_NAMES = frozenset(_PROFILES)` — the validatable set of profile name strings.
- `band_for(profile: str | None) -> Band` — case-insensitive, whitespace-tolerant lookup in `_PROFILES`; unknown/None/empty → `TALK` (safe default, never raises). Pure. Called from many places outside this cluster: `accounts.Accounts.validate`, `clip.render_account_cut`/`render_moment`, `crosspost.render_spec`, `moments.request_moment_hooks`, `persona_directives.*`, `persona_levers.build_catalog`, `prompts.moment_pick_prompt`, `studio.views_review._length_label`.

### frames.py — pure first-frame strength scoring (no subprocess itself; consumed by clip.py which owns the ffmpeg call)

- `_MIN_LUMA=16.0`, `_MAX_LUMA=244.0`, `_MIN_CONTRAST=24.0` (module consts) — degeneracy floors.
- `_YAVG`, `_YMIN`, `_YMAX` (compiled regexes) — parse ffmpeg `signalstats` filter output lines.
- `parse_signalstats(text: str) -> tuple[float,float] | None` — extracts `(luma, contrast)` = `(YAVG, YMAX-YMIN)` from raw ffmpeg text; `None` if any of the three keys is missing. Pure. Called by `clip._probe_frame_strength`.
- `parse_sharpness(text: str) -> float | None` — extracts YAVG alone as a relative sharpness proxy (mean edge energy of a Laplacian-convolved frame). Pure. Called by `clip._probe_frame_sharpness`.
- `frame_strength(*, luma, contrast, sharpness=None) -> float | None` — returns `None` if luma is outside `[MIN_LUMA, MAX_LUMA]` or contrast below `MIN_CONTRAST` (degenerate frame, rejected). Without `sharpness`, returns `contrast` directly (legacy contrast-only path). With it, returns `round((contrast * max(sharpness,0))**0.5, 4)` — geometric mean, demoting busy-but-soft frames. Pure. Called by `pick_strongest`.
- `pick_strongest(candidates: list[dict]) -> dict | None` — scores every candidate via `frame_strength`, filters out `None`s, sorts by `(-strength, -scene_score, earliest_t)`, returns the winner or `None` if nothing clears the floor. Pure. Called by `clip.pick_visual_start`.

### audio_energy.py — pure ffmpeg-astats RMS parsing (no subprocess call itself; `signals.detect_signals` owns it)

- `_PTS`, `_RMS` (compiled regexes) — parse `pts_time:` and `lavfi.astats.Overall.RMS_level=` lines.
- `_RMS_FLOOR_DB=-50.0`, `_RMS_CEIL_DB=-5.0` (module consts) — dBFS-to-[0,1] mapping band.
- `energy_cmd(src: str) -> list[str]` — builds the ffmpeg astats command list (does not execute it): `ffmpeg -hide_banner -i <src> -af astats=...,ametadata=print... -f null -`. Pure command-builder. Called by `signals.detect_signals` (which actually shells it).
- `parse_energy(text: str) -> list[dict]` — pairs each `pts_time:T` line with the *next* RMS_level reading into `[{"t": float, "rms": float}]`; treats `-inf` as literal `float("-inf")`; skips unpaired/garbled lines silently (never raises — ffmpeg text is "semi-trusted"). Pure. Called by `signals.detect_signals`.
- `rms_to_strength(rms_db: float) -> float` — linear-maps dBFS into `[0,1]`, clamped; `-inf` → `0.0`. Pure. Called by `signals.apply_energy`.

### vocals.py — Demucs vocal isolation, fail-open by contract

- `_DEMUCS_TIMEOUT = 1800.0`, `_DEFAULT_MODEL = "htdemucs"` (module consts).
- `_demucs_env() -> dict` — copies `os.environ`, sets `SSL_CERT_FILE`/`REQUESTS_CA_BUNDLE` to certifi's bundle if not already set (works around macOS framework Python's inability to verify TLS certs when demucs first downloads its model). Wrapped in a bare `try/except Exception: pass` if `certifi` import fails — this is the one broad-except in the cluster, but it's a deliberate best-effort env decoration, not error-swallowing on a critical path (see Anomalies).
- `demucs_cmd(audio_path, out_dir, *, model=_DEFAULT_MODEL) -> list[str]` — builds `["demucs", "--two-stems=vocals", "--mp3", "-n", model, "-o", out_dir, audio_path]`. Pure command-builder. Called by `isolate_vocals`.
- `isolate_vocals(audio_path, out_dir, *, model=_DEFAULT_MODEL) -> str` — **shells `demucs`**, timeout 1800s, using `_demucs_env()`. Fail-open on `FileNotFoundError`/`OSError`/`TimeoutExpired` → returns the original `audio_path` unchanged. Fail-open on nonzero return code → same. On success, returns the path to `<out_dir>/<model>/<stem>/vocals.mp3` if it exists, else falls back to the original path. **Never raises.** Called by `transcribe._produce_transcript`.

### intro_match.py — LLM-vision intro-asset pairing gate (post-clip enrichment, not raw ingest)

- `MATCHER_VERSION = "v1"` (module const, bump-to-reseed).
- `_INTRO_AWAITING = awaiting("intro_tease")` (module const, resolves to `"clean_awaiting_strategy:intro_tease"`).
- `_UNUSABLE_SOURCE_STATES = (retired, error, discovered)` (module const).
- `_candidates(led) -> list[Source]` — filters `led.sources` to `origin_kind == "third_party"` and not in `_UNUSABLE_SOURCE_STATES`, sorted by id for stable gate keys. Pure read.
- `_TERMINAL_MOMENT_STATES = (retired, error)` (module const).
- `_reserved(led) -> list[Moment]` — moments the router reserved for intro_tease (`hook_strategy == _INTRO_AWAITING`), excluding terminal states, sorted by id. Pure read.
- `_gate_key(m, cands) -> str` — hashes `("intro_match", moment.id, *candidate_ids, MATCHER_VERSION)` via `ids._hash`. Pure.
- `_frames(led, cfg, m) -> list[str]` — resolves the moment's source; if the file doesn't exist on disk, returns `[]` (fail-open, no ffmpeg spawn on an absent path); else calls `keyframes.extract_keyframes` (shells ffmpeg, outside this cluster) for 3 frames in the moment's window.
- `_thumb(cfg, s) -> str | None` — for an image-source candidate returns the path itself; for a video, extracts one keyframe near the head via `keyframes.extract_keyframes`; returns `None` if the file is absent (fail-open).
- `request_intro_match(led, cfg) -> Ledger` — no-op unless `cfg.intro_tease` and `cfg.responder_mode == "llm"`; no-op if no third-party candidates or no reserved moments exist. Otherwise builds a candidate payload (asset_id/origin_kind/label/thumbnail) and, per reserved moment, writes an agent gate request (`agentstep.write_request`) keyed by `_gate_key`, unless a request already exists for that key (`latest_request_id` check — idempotent, avoids invalidating an in-flight/answered gate). **Side effect**: writes a request control file; shells ffmpeg transitively via `_frames`/`_thumb`. Called by `pipeline._stage_structural_hooks`.
- `ingest_intro_match(led, cfg) -> Ledger` — no-op unless intro_tease+llm enabled, or no candidates exist. For each reserved moment, reads the matcher's response (`agentstep.read_response`, typed as `IntroMatchDecision`); `None` (no response yet, or corrupt — logged by agentstep) → skip, moment stays unmatched. Otherwise filters pairings to only those naming a real candidate asset id and carrying non-empty `tease_text`, sorts by `(-fit_score, asset_id)`, and writes them onto `led.moments[m.id].intro_matches`. **Side effect**: mutates ledger moment objects in place. Called by `pipeline._stage_structural_hooks`.

### transcribe.py — Whisper/faster-whisper transcription with vocal-isolation preprocessing

- `_WHISPER_TIMEOUT = 2700.0`, `_PREWARM_TIMEOUT_FACTOR = 1.5` (module consts).
- `_whisper_timeout(duration_seconds) -> float` — `max(_WHISPER_TIMEOUT, duration*1.5)` if duration known, else the floor. Pure.
- `_cached_models() -> list[str]` — lists `.pt` stems under `~/.cache/whisper` (or `$XDG_CACHE_HOME/whisper`). Read-only filesystem probe.
- `_resolve_model(model: str) -> str` — imports `whisper` to get `available_models()` (any import failure → returns `model` unchanged); if the requested model isn't a known name, falls back to `"turbo"` or the first known model; if the resolved model isn't cached on disk, tries to substitute a cached one (preference order: turbo, large-v3, medium, small, base, tiny) to avoid triggering an impossible download in air-gapped/CI environments; if nothing is cached, keeps the requested name and lets whisper attempt the download.
- `real_transcript_signal(transcript: list[dict]) -> bool` — a test-support helper proving a *real* whisper run occurred (not that any specific word survived): requires at least one segment with numeric `start`/`end` where `end > start`, AND the joined text has ≥4 alphabetic (Unicode-aware, EN+AR) word tokens. Pure, vocoder-agnostic. No repo callers (test-only utility per call graph).
- `whisper_cmd(src, out_dir, model="turbo") -> list[str]` — builds the legacy `whisper` CLI invocation with `--word_timestamps True`, `--output_format json`. Pure. Called by `_produce_transcript`.
- `_fw_available() -> bool` — probes whether `faster_whisper` is importable; any exception → `False`. No I/O beyond the import attempt.
- `fw_cmd(src, out_dir, model, language="") -> list[str]` — builds `[sys.executable, "-m", "fanops._fwrun", "--model", model, "--language", language, "--output_dir", out_dir, src]`. Pure. Called by `_produce_transcript`.
- `_segment(s: dict) -> dict` — normalizes one whisper segment to `{start, end, text}`, optionally including `words` if it's a well-formed list of `{word,...}` dicts (schema-guarded so malformed word data can't poison the caption overlay). Pure.
- `_adopt_cached_transcript(led, source_id, cached: Path) -> bool` — reads and parses a prior whisper JSON output, populates `led.sources[source_id].transcript`/`.language`/`.meta["transcribed"]`, calls `led.set_source_state(source_id, transcribed)`. Returns `False` (no mutation applied) on `JSONDecodeError`/`OSError` or on `KeyError`/`TypeError`/`AttributeError` during application — lets the caller fall through to a real (re-)run. **Side effect**: mutates the ledger Source in place, on success only.
- `transcribe_source(led, cfg, source_id, *, model=None) -> Ledger` — the main entry point. Idempotent short-circuit if `src.meta.get("transcribed") is True`. Fast path: checks for a deterministic cached JSON path (`agent_io/transcripts/<stem>.json`) and adopts it without any locking if present and valid. Otherwise acquires a per-`(stage="transcribe", key=source_id)` `stage_lock` (NOT the ledger flock — a deliberate M1 architecture change so concurrent sources don't serialize against each other), re-checks the cache inside the lock (closes a TOCTOU race between two producers for the same source), and if still absent, delegates to `_produce_transcript`. **Side effects**: mkdir, ledger mutation, lock acquisition. Called by `pipeline._stage_source_to_moments`, `produce._produce_one`.
- `_produce_transcript(led, cfg, source_id, src, out_dir, model) -> Ledger` — the slow path, only reachable from inside the stage lock. If `cfg.isolate_vocals`, calls `vocals.isolate_vocals` (shells demucs); on success, moves the resulting vocals mp3 to `<out_dir>/<source_stem>.mp3` via `Path.replace` — on move failure (cross-device etc.) falls back to the raw source path rather than the demucs output path (preserves the deterministic cache-lookup contract). Picks the engine: faster-whisper (`fw_cmd`) if `_fw_available()`, else legacy `whisper_cmd`, both duration-aware model selection via `cfg.asr_model_for`/`cfg.whisper_model_for`. **Shells whisper or faster-whisper** (`subprocess.run`, `timeout_s = _whisper_timeout(src.duration)`). Error handling is fully explicit and typed at every failure point — binary absent → `SourceState.error` + `error_reason` (toolchain missing); timeout → `SourceState.error` + timed-out reason; no JSON produced → `SourceState.error` + reason with stderr tail; malformed JSON (`JSONDecodeError`/`KeyError`/`TypeError`/`AttributeError`) → `SourceState.error` + reason. On success, sets `src.transcript`, `src.language`, `src.meta["transcribed"] = True`, `led.set_source_state(source_id, transcribed)`. **No silent failures** — every error path sets a typed, readable `error_reason` on the ledger row rather than raising or swallowing.

## Audio analysis: how bands/frames/audio_energy/vocals/intro_match combine to inform clip selection

These modules are **not** wired directly to each other — they're independent pure-analysis libraries, each consumed by a different downstream orchestrator (mostly `clip.py`/`signals.py`, outside this cluster):

- **`vocals.py`** runs first, chronologically, inside `transcribe._produce_transcript` — it strips the instrumental via Demucs so Whisper transcribes lyrics rather than gibberish-over-a-beat. Its output (`vocals_isolated` meta flag) is later read by `framing.classify_window` (outside this cluster) to treat non-speech windows as MUSIC rather than silence.
- **`audio_energy.py`** is pure ffmpeg-astats parsing consumed by `signals.detect_signals`/`signals.apply_energy` to turn loudness (RMS dBFS) into a normalized `[0,1]` strength commensurable with scene-cut scores — this is what lets a musical "drop" out-rank a mere scene cut when picking impactful moments.
- **`frames.py`** is pure ffmpeg-signalstats parsing consumed by `clip.pick_visual_start`/`clip._probe_frame_strength`/`_probe_frame_sharpness` to choose the strongest opening still frame for a rendered clip (the muted-autoplay first-frame lever) — degenerate (near-black/blown/flat) frames are rejected by a luma/contrast floor.
- **`bands.py`** defines the target clip-length window (`Band(lo, hi)`) per content-type/operator profile (talk/song/short/medium/long); `band_for` is consulted by `clip.render_moment`/`render_account_cut` to bound render length, and by `prompts.moment_pick_prompt`/`persona_directives.*` to inform the moment-selection LLM prompt of the target length.
- **`intro_match.py`** is unrelated to the audio-signal chain — it's a separate LLM-vision gate that pairs a clean clip with a third-party intro asset for a "wait for it" tease, gated behind `cfg.intro_tease` + `responder_mode=="llm"`, fully fail-open.

So the composition point is `signals.py`/`clip.py` (outside this cluster): they import the pure scoring primitives from `audio_energy.py`, `frames.py`, and consult `bands.py` for length targets, to jointly decide where a moment starts/ends and which frame it opens on. `vocals.py` acts earlier in the pipeline (transcription quality), `intro_match.py` acts later (post-render pairing).

## Anomalies found

None of the "bad" patterns the task asked to hunt for (bare `except: pass`, silent error swallowing, TODO/FIXME/XXX, unvalidated path-traversal) are present as *defects* — this codebase is unusually disciplined about typed errors and fail-soft-with-a-breadcrumb. Specific notes:

1. **No TODO/FIXME/XXX comments** anywhere in the 9 files (`grep` confirmed zero matches).
2. **No bare `except:` or `except: pass`** in the 9 files (grep confirmed zero matches). The closest pattern is a deliberate best-effort decoration:
   - `src/fanops/vocals.py:35-36` — `_demucs_env()`: `try: import certifi; ... except Exception: pass`. This swallows *any* exception from importing/using certifi, not just `ImportError`. It's low-risk (only decorates a subprocess env dict with SSL cert paths; a failure here just means demucs might hit its own TLS-cert issue later, which is itself already fail-open via `isolate_vocals`'s broader try/except) but it is a genuinely overbroad except clause worth narrowing to `ImportError` for legibility.
   - `src/fanops/discover.py:59-62` — `candidate_meta()`: `except Exception as e: logging.getLogger(...).debug(...)`. This is intentionally broad (module docstring explicitly frames discovery as "fail-soft — list it anyway"), and it does log (at debug level) rather than truly swallow. Not a defect, but the debug level means a real `probe_dimensions` bug could go unnoticed in production logs (info/warning would be more discoverable) — worth flagging as a legibility gap, not a correctness bug.
3. **Path-traversal-relevant validation** — `discover.intake()` at `src/fanops/discover.py:140-142` explicitly checks `src is None or not src.exists() or src.is_symlink()` before copying, specifically to avoid copying a symlink target out of bounds (comment cites "ECC fix #9"). `ingest.ingest_drops` and `ingest.scan_local` both explicitly skip `f.is_symlink()` before any copy/probe (also citing "ECC fix #9"), and `ingest_drops` further checks `archive in f.resolve().parents` to avoid re-scanning its own archive directory. These are *hardening* patterns already in place, not gaps — no unvalidated path-traversal surface found in this cluster.
4. **Dead-code candidate**: `ingest.download_source` — `call_graph.json` shows `called_by_in_repo: []` (no repo caller besides tests, per its own docstring: "kept for any direct caller/test"). The CLI's real `pull` path (`cli.cmd_pull`, outside this cluster) calls `download_url` and `ingest_drops` separately rather than through this wrapper. Confirm before removing — the docstring explicitly says it's intentionally retained for direct callers/tests, so this is very likely deliberate, not orphaned.
5. **Dead-code candidate (weaker)**: `discover.discover` and `discover.intake` both show `called_by_in_repo: []` in the call graph. This is very likely a call-graph limitation (they're invoked via a CLI dispatch table / `fanops-discover` skill using string-based subcommand routing rather than a direct Python call the AST-based graph can trace), not actual dead code — the module's own docstring frames this as a first-class two-step review workflow. Flagging for verification against `cli.py`'s dispatch table rather than asserting it's genuinely unreachable.
6. **`transcribe.real_transcript_signal`** — `called_by_in_repo: []`. This is explicitly documented as a test-support helper ("Used by the real-tooling E2E") — expected to be called only from test files, which the call graph (scoped to `src/fanops/`) wouldn't capture if tests live under `tests/`. Not a defect.
7. **Fail-open-by-design surface worth naming explicitly** (not a bug, but a design choice with real consequences): `vocals.isolate_vocals`, `discover.make_thumbnail`, and the ffprobe-timeout branch inside `ingest._run_ffprobe` all convert hard failures (absent binary, hang, nonzero exit) into soft degraded-but-successful returns rather than propagating. This is consistent, documented, and intentional throughout (explicitly contrasted against `download_url`/`transcribe._produce_transcript`'s error-surfacing whisper/yt-dlp paths, which correctly do NOT fail open because a missing transcription or download is a real pipeline defect, not a nice-to-have enhancement). No inconsistency found between the two postures — the split (enhancement-quality subprocess → fail open; core-pipeline-input subprocess → fail loud with typed error) is applied correctly and uniformly across all 9 files.
