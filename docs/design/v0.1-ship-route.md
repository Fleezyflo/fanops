# v0.1 prototype ship route

Binding acceptance checklist for the v0.1 prototype milestone. Command-accurate; no marketing prose.

## §0 · Acceptance demo (7 steps)

### 1 · Install

```bash
python3.12 -m venv .venv && source .venv/bin/activate
pip install -e '.[dev,studio,transcribe,framing]'
./scripts/setup-hooks.sh
```

### 2 · Doctor

```bash
fanops doctor
```

**Success:** exits `0` (all checks green).

### 3 · Studio + Go Live intake (S04)

```bash
fanops studio
```

Open http://127.0.0.1:8787 → **Go Live** tab:

1. **1 · Connect Postiz** — paste Postiz URL + API key → **Save & test** (green *connected …*).
2. **2 · Add an account** — enter handle, tick platform checkboxes → **Add account**.
3. **3 · Map channels & tune accounts** — **Refresh from Postiz** (or **Discover channels (Postiz + Zernio)**), pick integration per channel → **Save channel mappings**; route each channel to a scheduler backend if needed.
4. **4 · Readiness** — checklist green; **5 · Go live** channel readiness matrix shows every active channel **ready** (mapped ✓, creds ✓, persona ✓, window ✓, no blocker).

### 4 · Upload (S02)

**Run** tab:

1. **Add footage** — pick one or more video files (chunked resumable upload for large files).
2. **Add to queue** — catalogue into the pipeline (upload auto-catalogues on submit; Finder drops land on the next **Make clips** press).

**Success:** *N saved, 0 skipped* (or equivalent added count) and sources count increases.

### 5 · Prepare + Review focus (S07)

**Run** tab → **Make clips** (end-to-end cut + captions).

**Review** tab:

- Bare `/review` with multiple accounts → account-first picker ("Which account … to review").
- Bare `/review` with one account → auto-focus redirect (`?account=…&view=account&focus=1`).
- Check posts → **Approve selected** (moves to Schedule; not live yet).

### 6 · Schedule → publish

If not already live: **Go Live** tab → tick **⚠ I understand this publishes to REAL accounts** → **GO LIVE — publish for real**.

- **Schedule** tab → per-row **Publish now**, or
- **Run** tab → **Prepare everything** with **⚠ publish to REAL accounts** confirm ticked.

**Success:** post ships to the real account; published count increases.

### 7 · Metrics + S06 rotation

**Posted** tab:

- Shipped post shows live URL.
- Metrics arrive (timestamp on row after platform reports).

**S06 rotation evidence:** two consecutive posts for the same account show **differing** hashtag tag lines (Posted **Tag exposure** summary or caption hashtag line differs between the two most recent posts for that handle).
