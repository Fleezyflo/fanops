## 1. What this is

An autonomous fan-account engine for Moh Flow (bilingual EN/AR rapper). It ingests his videos, decides which moments are worth posting (transcript + audio/scene signals → an agent decision with a recorded reason), cuts platform-ready clips (per-account length/framing/hook differentiation; a dynamic content-adaptive reframer) with agent-written EN/AR captions, cross-posts to every fan account × platform (Postiz for IG, Zernio for TikTok; staggered for opsec, with a subtle non-synchronized artist @mention), then pulls real performance back (IG via the Meta Graph) to make more of what works. **FanOps Studio** (`fanops studio`) adds a local web cockpit for the operator's editorial job: watch the rendered clips, tweak captions, approve/reschedule/snooze the queue, read per-variant lift, manage personas + hashtag corpora, and flip go-live — without touching the autonomous machinery. See `README.md` (front door), `MohFlow-FanOps/00_control/RUNTIME.md` (operating loop + production seams), `MohFlow-FanOps/00_control/RISK.md` (operator risk-acceptance for the multi-account model), `MohFlow-FanOps/00_control/context.md` (functional — injected as agent guidance).

## 2. Architecture

- **Unit chain:** `Source → Moment → Clip → (Render) → Post`, over one git-versioned JSON ledger (`00_control/ledger.json`, atomic temp+os.replace write under a file lock, day-bucketed + persisted). `Render` is a per-account child of `Clip` (the authoritative pointer for an account's own cut).
- **Identity:** content-addressed SHA-based ids everywhere (`ids.py`: `make_id`/`child_id`/`surface_key`) — NEVER builtin `hash()` (the #1 v1 bug; cross-process stability is proven by a subprocess idempotency test).
- **Agent gates:** generative steps (`decide_moments`, `moment_casting`, `write_captions`, hooks) cross a file contract in `04_agent_io/requests/` — code writes a `*.request.json` (stamped with a `request_id`), the agent/responder writes a validated `*.response.json` echoing that id, code resumes. Stale responses can never be applied (request_id correlation).
- **Two responders** (`responder.py`): `ManualResponder` (no-op; a human/cron writes responses) vs `LlmResponder` (`FANOPS_RESPONDER=llm`; **autonomous via `claude -p`** — shells the Claude Code CLI through `llm.claude_json` with committed `prompts.py` templates + the gate's pydantic JSON schema, validates against the gate's model, **per-request quarantine**). Needs `claude` on PATH (authenticated); model is injectable for tests.
- **Poster backends** (`post/providers.py`): `dryrun` (default), `postiz` (`PostizPoster`, the live IG path), `zernio` (`ZernioPoster`, TikTok). Blotato's `rest`/`mcp` posters were DELETED in Leg 1.
- **Pipeline** (`pipeline.py` `advance()`): per-unit error quarantine (one bad source/moment/clip → `error` state, never wedges the pass); it self-ingests, so the launchd daemon (`fanops daemon` = packaging of `fanops run`) is the hands-off driver. **CLI** (`cli.py`): `status/ingest/pull/advance/respond/track/adjust/gc/run/digest` + recovery verbs + `discover`/`intake` + `hashtags` + `daemon`/`autopilot`/`cutover`/`doctor` + **`studio`** (lazy-imported web UI).
- **Studio** (`studio/`): `views*.py` pure read-models (Flask-free) → `actions*.py` lock-safe mutations → `app*.py` Flask factory (tab routes — Home/Review/Schedule/Posted/Personas/Go-Live + variant-aware `/media/<post_id>` + POST mutation routes). Templates + vendored HTMX; no JS build step.
- **State machines** (`models.py`): separate `SourceState`/`MomentState`/`ClipState`/`PostState`. A crossposted post is BORN `awaiting_approval` (no auto-publish — the operator promotes it to `queued`); `held` (brand-risk), `needs_reconcile`, and `retired` are first-class.
- **The learning loop** (`track`→`adjust`→`variant_amplify`/`p4_dim_bias`/`timing_bias` + casting's brief-time reach prior): every actuator is amplify/bias-only (C1 — never retire/cascade/track), gated on `learning_validated` (auto-unfreezes on the first real non-degraded live metric) + `p4_unlocked`, each behind a default-OFF kill switch. Codemap: `docs/CODEMAPS/insights-culmination.md`.

## 3. Gotchas / non-obvious invariants

- **`--base-time` is the SCHEDULE ANCHOR, not the publish cutoff.** crosspost staggers posts to times AFTER base-time; `publish_due(now=None)` publishes whatever is due as of REAL wall-clock now. To publish in the same pass (backfill), pass a `--base-time` in the past.
- **No auto-publish (HARD rule).** A crossposted post is born `awaiting_approval`; `publish_due`/`publish_now` iterate only `queued`, so NOTHING publishes (even live / on the daemon) until the operator approves it in the Studio Review tab. `queued` = "approved + scheduled"; `rejected` = operator discard.
- **Conservative retirement:** `adjust` amplifies top `--winner-pct` (0.3) but retires only the bottom `--retire-pct` (0.2) AND below `--lift-floor` (20.0). Retiring a clip also retires its moment.
- **Two surfaced-but-auto-unrecovered stuck states** (in the digest + counts, no silent loss): `submitting` (mid-publish crash) and **`needs_reconcile`** (ambiguous 5xx/timeout — NOT re-POSTed, may be live). `needs_reconcile` arises under a live backend, never `dryrun`.
- **`dryrun` is a DISTRIBUTION BOUNDARY, not a publisher** (dryrun-boundary M1+M2, PRs #271/#272): a dryrun (not-live) post is built + approved + scheduled and then HALTS `queued` at the `publish_due`/`publish_post` seam — it never enters distribution, never becomes a phantom `published` row. `DryRunPoster` is a PREVIEW writer (writes the would-send `05_scheduled/<id>.json` sidecar, 0o600) and stamps NO `state`/`submission_id`/`public_url`. The old `dryrun_<post_id>` synthetic-id money loop is GONE (there is no offline dryrun learning loop). M3 (in progress) deletes the now-inert readers (`dryrun://` scheme, `is_phantom_published`, `doctor-fix-ghosts`, the ledger migration-on-read back-fill).
- **tz-correctness for timing:** `publish_hour`/`publish_dow` are stamped AND ranked in the operator-local tz (`cfg.operator_tz`, fails CLOSED to UTC on an unknown zone) — a UTC-bucketed hour is noise for a single-region audience.
- **Whisper model:** default `turbo`; falls back to the best cached `~/.cache/whisper/*.pt` when the requested model can't be downloaded. In THIS env only `tiny` is cached. Music ASR uses the `[asr]` extra (Demucs + faster-whisper, lazy + fail-open to the legacy whisper CLI).
- **The dynamic reframer** (`framing.py`, `FANOPS_SMART_FRAMING` default ON) is a STATIC locked-off crop per shot (jitter-free by construction — a per-frame chase was tried and reverted); OpenCV is the optional `[framing]` extra, fail-open at every step → centered crop.
