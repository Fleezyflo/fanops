## 2. What `RC-9` actually is

`Post` carries a `@model_validator(mode="after")` ([models.py:357](src/fanops/models.py:357)) that **raises**
when `state ∈ {published, analyzed}` and `public_url` is blank.

**Executed against pydantic 2.13.4:**

| Door | Result |
|---|---|
| `Post(**v)` — construction | ✅ **RAISES** |
| `model_copy(update={"state": published})` | 🔴 **NO RAISE** |
| `post.state = PostState.published` — `setattr` | 🔴 **NO RAISE** *(`Post` has no `validate_assignment`)* |
| `model_dump()` of the bad row | 🔴 **serializes cleanly** |
| **the next `Ledger.load` of that row** | 🔴 **RAISES → `ControlFileError` → THE ENTIRE LEDGER IS UNLOADABLE** |

> 🔴 **The failure mode is not a rejected write. It is a LOAD-TIME POISON PILL.** The bad row **saves
> successfully** and then **bricks the daemon and every Studio page at once**, on the next process start.

**The property survives via FOUR INDEPENDENT MANUAL CALL-SITE GUARDS:**

| Door | Its real guard |
|---|---|
| `_publish_one` | `if (post.public_url or "").strip():` — [run.py:305](src/fanops/post/run.py:305) |
| `actions.mark_published` | rejects a blank url up front — [actions.py:268](src/fanops/studio/actions.py:268) |
| `cli.cmd_resolve` | `--url` **required** for terminal states — [cli.py:384](src/fanops/cli.py:384) |
| `DryRunPoster.publish` | post-M2 it sets **no state at all** — [dryrun.py:39](src/fanops/post/dryrun.py:39) |

**And `models.py`'s own docstring claims the invariant is *"bound at the type level so no door can produce the
ghost row."* 🔴 THAT IS FALSE FOR FOUR OF THE FIVE DOORS IT NAMES.** Two code comments additionally assert
*"Pydantic re-validates on serialization."* **It does not.**

> **This is the codebase's signature defect, in its purest form: the doc names a mechanism that does not exist,
> while the property survives via a different one.**

## 3. The slice

### Test 1 — `test_no_unguarded_door_to_a_terminal_post_state`

**An AST policy test over `src/fanops/`.**

- Enumerate **every** write site that can set `Post.state` to `published` or `analyzed`. **Cover the generic and
  dynamic shapes a literal grep cannot see** — this is the `C2-SC-1` lesson, and it is binding here:
  - `p.state = PostState.published` *(literal attribute assignment)*
  - `p.state = <var>` *(dynamic)*
  - `PostState(<str>)` *(constructed from a string — e.g. `cli.py:395`)*
  - `setattr(p, <var>, v)` *(e.g. `run.py:357`, the FINALIZE merge — `_NET_POST_FIELDS` **contains `"state"`**)*
  - `model_copy(update=<literal dict>)` and `model_copy(update=<var>)` *(e.g. `reconcile.py:725`)*
  - `Post(state=...)` constructor kwargs, literal **and** variable
  - **enum-valued keyword defaults** *(the `FIND-007` trap — there is exactly **one** in the tree, `born_state`)*
- **Assert each such site is accompanied by an explicit non-empty `public_url` guard.**
- 🔴 **The FOUR known doors are the BASELINE — exactly like the swallow ratchet. A FIFTH makes CI RED.**

### Test 2 — `test_model_copy_does_not_bypass_a_raising_validator_undetected`

**Pin the real pydantic behaviour, executably.**

For every model with a raising validator, assert the **current (bypassing)** behaviour and **name it**:

| Invariant | ctor | `setattr` | `model_copy` |
|---|---|---|---|
| `Post`: published requires `public_url` | raise | **BYPASS** | **BYPASS** |
| `Post.account` canonicalization | raise | **BYPASS** | **BYPASS** |
| `Moment.affinities` canonicalization | raise | **enforced** | 🔴 **BYPASS** *(despite `validate_assignment=True`)* |
| `Moment.segments` validity + envelope | raise | **enforced** | 🔴 **BYPASS** |

> **Why pin a *bug*?** Because **two code comments assert the opposite**, and a pydantic upgrade that *changed*
> this behaviour would silently alter the safety posture of ~57 call sites. **This test makes the real behaviour
> executable rather than folkloric** — and it catches the upgrade.

## 5. Preservation boundary

- 🔴 **DO NOT FIX `RC-9`.** Do not add `validate_assignment` to `Post`. Do not build a validated-mutation API. Do

  with `validate_assignment=True`, and `cast_add`/`cast_remove` are correct **only** because of that setattr
  (`COUP-07`). A non-canonical `"@Foo"` would make that moment post **nowhere**, silently.

  DEFAULT.** Setting `forbid` — *a change that looks like tightening* — turns a forward-rolled ledger into a hard
  `ControlFileError` (`SHIM-005`).
